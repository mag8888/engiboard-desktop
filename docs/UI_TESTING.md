# UI / визуальное тестирование

Автоматическая проверка, что фронтенд EngiBoard **работает, кликается, включается и выключается** — особенно на Windows.

## Что это

Playwright-сьют, который грузит **реальный `dist/index.html`** (тот же фронтенд, что встроен в десктоп-сборку) в браузерном движке и реально кликает по элементам: создаёт задачи, открывает/закрывает меню, переключает фильтры, тогглит баг/архив/пин/тёмную тему, листает презентацию и т.д. Каждый шаг проверяется по состоянию приложения и снимается скриншот.

### Почему именно так покрывает Windows

Десктоп-сборка под Windows рендерит UI в **WebView2** (движок Edge/Chromium). Сьют на Windows запускается в канале **`msedge`** — это тот же движок. Поэтому CSS/JS-баги и поведение кликов/тогглов воспроизводятся 1:1 с реальным приложением, без запуска нативной оболочки.

**Что НЕ покрывает:** нативные функции Tauri (захват экрана, machine fingerprint, tray, drag файлов, лицензия) — для них нужен отдельный `tauri-driver` E2E (TODO).

## Запуск локально

```bash
npm install
npx playwright install chromium     # один раз
npm run test:ui:chromium            # mac/linux — chromium
# или весь сьют (chromium + edge, если Edge установлен):
npm run test:ui
```

Скриншоты каждого шага — в `screenshots/`. HTML-отчёт:

```bash
npx playwright show-report
```

## Запуск на Windows (CI)

Workflow `.github/workflows/ui-test.yml` гоняет сьют на **windows-latest в Edge** при каждом push/PR, затрагивающем `dist/`, `tests/`, конфиг. Можно запустить вручную: вкладка **Actions → UI smoke (Windows / Edge·WebView2) → Run workflow**.

Артефакты прогона (скачать со страницы run):
- `ui-screenshots-windows-edge` — скриншоты всех шагов (визуальная проверка).
- `playwright-report-windows-edge` — HTML-отчёт + трейсы упавших тестов.

## Что проверяется (15 сценариев)

| # | Сценарий | Проверка |
|---|---|---|
| 01 | Загрузка | шапка, сайдбар, задачи рендерятся; ноль ошибок консоли |
| 02 | Создание задачи | ввод снизу + Enter → задача с текстом в чате |
| 03 | Меню статуса | открыть → выбрать → закрылось |
| 04 | Меню ассайни | повторный клик закрывает; клик вне закрывает |
| 05 | Эксклюзивность меню | открыто только одно из статус/ассайни/kebab |
| 06 | Мультифильтр | 2 статуса активны; «All» сбрасывает |
| 07 | Тип «баг» | пометка → бейдж BUG; чип Bugs вкл/выкл |
| 08 | Архив | toggle меняет aria-pressed / состояние |
| 09 | Пин | появляется секция Pinned |
| 10 | Тёмная тема | включается и выключается |
| 11 | Сортировка | переключение на Status |
| 12 | Презентация | открыть → next/prev → закрыть |
| 13 | Галерея в презентации | клик по миниатюре меняет вьюер |
| 14 | Dashboard | кнопка New project; навигация назад |
| 15 | Переключение вида | список/развёрнутый без ошибок консоли |

## Добавить новый тест

Дописать в `tests/ui/smoke.spec.js`. Хелпер `ensureApp(page)` грузит demo-режим и встаёт на проект. Для шагов, где элемент в скролл-контейнере с `contain:paint` (например поле новой задачи внизу), дёргать реальный обработчик через `page.evaluate(...)`.

## Важно: источник фронтенда

Живой фронтенд — **`dist/index.html`** (на него указывает `frontendDist` в `tauri.conf.json`, его же грузят тесты). Корневой `index.html` устарел и не используется — **не запускать `npm run sync-dist`** (он затрёт `dist/` старой версией).
