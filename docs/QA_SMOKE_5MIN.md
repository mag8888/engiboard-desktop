# Smoke 5 минут — EngiBoard

Минимальный проход для уверенности "не сломали базовое". Цель: за 5 минут поймать критику. Идти строго по порядку.

---

## 0. Запуск (30 сек)

- [ ] Приложение открылось без диалога об ошибке
- [ ] Видны проекты в сайдбаре
- [ ] Заголовок проекта читаем, нет "undefined" / голых ID

## 1. Создание задачи (1 мин)

- [ ] Cmd-N или кнопка "+ New task" — задача появляется
- [ ] Кликнуть на статус — открывается выпадашка
- [ ] Поменять статус на Done — цвет пилюли поменялся
- [ ] Cmd-Z — статус откатился

## 2. Скриншот + редактор (1.5 мин)

- [ ] Capture или paste картинки в Before — превью появилось
- [ ] Клик по превью — открыт редактор аннотаций
- [ ] Клавиша A — выбран инструмент Arrow
- [ ] Нарисована стрелка
- [ ] Save — вернулись на главную, картинка обновилась со стрелкой

## 3. Чат (30 сек)

- [ ] В строке чата задачи написано "test"
- [ ] Enter — сообщение появилось, поле очистилось
- [ ] Hover на своё сообщение → крестик удаляет

## 4. Present + deadline (1 мин)

- [ ] Двойной клик по задаче — открылся Present mode
- [ ] Esc — закрылся
- [ ] В контекстном меню задачи "Set deadline" — выбрана дата
- [ ] В строке задачи появился чип с датой
- [ ] Открыт Present — чип с датой виден в шапке

## 5. Архив (30 сек)

- [ ] Иконка-кубик на карточке — клик
- [ ] Задача "исчезла" (или помечена как archived при включённом фильтре Archived)
- [ ] Cmd-Z вернул

---

## Если хоть один пункт ✗

- Открыть Console (если есть DevTools) → скопировать первую красную ошибку
- Заполнить issue по шаблону Bug report (`.github/ISSUE_TEMPLATE/bug_report.md`)
- Поставить лейбл `p0` если сломан весь раздел, `p1` если только конкретное действие
