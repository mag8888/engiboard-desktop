# EngiBoard — Продуктовые требования и тестовое покрытие

Версия документа: v0.1.158 (2026-06-08).
Аудитория: QA-тестировщик, разработчик, продакт.
Цель: единая точка правды о том, что и как должно работать. Каждая глава — это и описание поведения, и чек-лист для тест-кейсов.

---

## 1. Что за продукт

EngiBoard — настольный (Tauri/macOS + Windows) и веб (тот же HTML) трекер инженерных задач для бригад на стройке/производстве. Главная сущность — задача с двумя скриншотами (Before / After), статусом, исполнителем, дедлайном, чатом и аннотированными изображениями. Задачи группируются по неделям внутри проекта; проекты группируются по клиенту.

Облако: Supabase (Postgres + Realtime + Auth + Storage). Работает офлайн (LocalStorage), синхронизируется при подключении.

Платформы:
- macOS desktop (Tauri 2, WKWebView): DMG + .app
- Windows desktop (Tauri 2, WebView2): MSI + NSIS .exe
- Веб (любой Chromium-браузер; FF/Safari best-effort)

---

## 2. Глоссарий

| Термин | Значение |
|---|---|
| Задача (Task) | Строка с номером, статусом, чатом, Before/After-картинками |
| Неделя (Week) | Группа задач по году+номеру недели, формат `26W14` |
| Проект (Project) | Набор недель; принадлежит клиенту |
| Клиент (Client) | Группа проектов (свернутая в сайдбаре) |
| Present mode | Полноэкранный режим показа одной задачи с навигацией Prev/Next |
| Аннотация | Векторная фигура поверх скриншота (rect, arrow, text и т.д.) |
| Public link | Публичная ссылка на задачу с токеном, доступная без логина |
| Presence | Realtime-отображение, кто ещё открыл эту же задачу |
| Capture | Действие "сделать скриншот" (camera button / shortcut / paste) |

## 3. Аутентификация и роли

### 3.1 Регистрация и вход
- Email + пароль через Supabase Auth.
- Magic link не используется.
- При первом входе — попадание на свой профиль и список своих проектов.

### 3.2 Профили
- Аватар: инициалы из имени, цветной фон.
- Имя редактируется в Profile.
- Email менять нельзя.

### 3.3 Роли (RBAC)
| Роль | Может |
|---|---|
| owner | всё, включая удаление проекта, смену ролей |
| admin | редактировать всё в проекте, приглашать |
| editor | создавать/менять задачи, писать в чат |
| viewer | только смотреть, скачивать публичную ссылку |

Проверки QA:
- viewer не видит кнопок Capture, Archive, Delete на задачах
- viewer не может писать в чат
- editor не видит "Manage members"
- owner видит и может всё

## 4. Проекты и клиенты

### 4.1 Создание проекта
- Кнопка "+ New project" в сайдбаре открывает модалку: название, клиент (опц.).
- **Enter в поле названия = Create** (DB feedback v0.1.157 N6 — обязательно).
- Esc закрывает модалку, состояние не теряется при повторном открытии.

### 4.2 Группировка по клиенту
- Если у двух+ проектов один клиент — рисуется свёрнутая шапка `▸ Клиент (3)`.
- Состояние свёрнутости — per-user, LocalStorage ключ `eb_client_groups_collapsed`.

### 4.3 Удаление / переименование
- Контекстное меню по правой кнопке на проекте.
- Удаление с подтверждением, undo 10 секунд через тост.

## 5. Задачи — главная сущность

### 5.1 Поля задачи
- `n` — номер (CL-12, ATL-7), генерится по префиксу проекта + порядковый.
- `s` — статус (см. 5.2).
- `title` / `n` отображаются в `.title-text` и `.num`.
- `shot1` / `shot2` — Before / After (URL изображения).
- `chat` — массив сообщений.
- `notes` — заметки (видны в Present).
- `assignee` — id пользователя.
- `deadline` — ISO дата (опц.).
- `hidden` — архив (true/false).
- `pinned` — закреп вверху (true/false).
- `links` — массив `{id, url, title}`.

### 5.2 Статусы
| Ключ | Лейбл | Цвет | Семантика |
|---|---|---|---|
| 0 | Info | серый | информационная пометка |
| 1 | Done | зелёный | закрыто |
| 2 | Not Relevant | пурпурный | отметаем |
| 3 | Review | синий | "на потом" / на проверку |
| 4 | Info Required | жёлтый | нужно уточнение |
| 5 | Problem / In Progress | красный | важно и серьёзно |
| 6 | Upcoming | оранжевый | планируется |
| 7 | Blocked | сиреневый | заблокировано |

QA: смена статуса через клик по пилюле — открывает dropdown; смена статуса в Present mode — без перерисовки модалки (см. patchPresent).

### 5.3 Группировка по неделям
- Заголовок недели: `YYWNN  N done` (например `26W14  2 done`).
- Свёрнутость per-проект, LocalStorage `eb_collapsed_weeks_<projectId>`.
- Кнопка ✓ в правом углу шапки недели = завершить все задачи недели.
- Кнопка `»` справа = roll-forward открытых задач в следующую неделю.

### 5.4 Drag-drop задач
- Hold drag-handle `⋮⋮` слева — перетаскивание.
- Drop на верхнюю/нижнюю половину соседней строки — синяя полоса (`drag-over-top` / `drag-over-bottom`) и вставка туда.
- Drop на заголовок недели — перенос в эту неделю.
- Drop на "Pinned"-шапку **не** должен превращать `w` в `__pinned__` (защита от data corruption, исправлено в v0.1.153).

### 5.5 Архив (`hidden`)
- Иконка-кубик в шапке карточки задачи (`.card-archive`, видна всегда, не только в меню).
- Click — toggle. Тост "Archived" / "Restored" + Cmd-Z отменяет.
- `aria-pressed` отражает состояние (для скринридеров).
- Архивные задачи показываются только если включён фильтр "Archived" / chip в фильтр-баре.
- Архивный row: полупрозрачные диагональные полосы + бейдж "ARCHIVED" в правом углу. **Кнопка Unarchive остаётся при `opacity:1`** — не должна выглядеть disabled (фикс v0.1.158).

### 5.6 Пин (`pinned`)
- Меню задачи → Pin to top.
- Запиненные сортируются в самый верх над всеми неделями, отдельная шапка "Pinned".
- Пин-иконка filled при `pinned=true`.

### 5.7 Дедлайн (`deadline`)
- Меню задачи → Set deadline → date picker.
- Чип `[icon] 15 Jun` рядом с Assignee в каждой задаче и в Present mode.
- Просроченные — красный border у чипа.
- Меню задачи показывает текущее значение в названии пункта: "Change deadline (15 Jun)".
- Clear deadline = отдельный пункт меню, появляется только если дедлайн установлен.
- Изменение дедлайна **при открытом Present** обновляет чип без мерцания (patchPresent({deadline:true})).

### 5.8 Sort modes
- Sort menu: By week (default), By status, By assignee, By updated.
- Выбор сохраняется per-project в `eb_sort_<projectId>`.

### 5.9 Фильтры (chips в верхней панели)
- All, Info, Done, Not Relevant, Review, Info Required, In Progress, Upcoming, Blocked.
- Множественный выбор. Если выбран один — остальные не светятся.
- Поиск (`.search-input`) фильтрует по title, chat-content и notes.

### 5.10 Bulk-операции
- Click "Select" в правом верхнем углу — режим выбора.
- Чекбоксы появляются слева от каждой строки.
- Шифт+клик — выбрать диапазон.
- Bulk-меню: Change status, Archive, Delete, Move to week.

### 5.11 Контекстное меню (правая кнопка по задаче)
- Task info / History / Copy text / Copy public link / Duplicate / Pin / Insert template / Move to week / Set deadline / Clear deadline (если есть) / Delete.

## 6. Скриншоты (Before / After)

### 6.1 Способы добавить
- Кнопка-камера в шапке (`Capture`) — нативный screencapture (macOS) или xcap (Windows).
- Drag-drop файла прямо в Before/After-плейсхолдер.
- Cmd/Ctrl-V с буфера обмена.

### 6.2 Хранение
- В Supabase Storage bucket `task-screenshots`.
- При офлайне — base64 в LocalStorage, синхронизация при следующем подключении.

### 6.3 Замена / удаление
- Клик по существующему скрину открывает редактор аннотаций.
- В редакторе есть Delete photo.

### 6.4 Empty state
- Плейсхолдер: иконка камеры + "No after image — Capture, paste, or drop".

## 7. Редактор аннотаций

Открывается клик-на-скриншот или Capture.

### 7.1 Инструменты
| Шорткат | Инструмент |
|---|---|
| V | Select (двигать/удалять фигуры) |
| H | Hand (панорама) |
| R | Rectangle |
| C | Circle |
| A | Arrow |
| L | Line |
| P | Freehand pen |
| T | Text |
| B | Blur |
| Q | Comment pin |
| M | Marker (highlight) |

Важно: **клавиша D отключена** (была у инструмента Dimension, который скрыт в v0.1.157; кнопка убрана + шорткат снят в v0.1.158).

### 7.2 Опции
- Цвет (палитра 6 цветов + кастом).
- Толщина обводки.
- Стрелка: направление (one-way / two-way), filled head, размер.
- Shift — констрейн (горизонталь/вертикаль/45°, идеальный квадрат/круг).
- Esc — снять выделение.
- Delete/Backspace — удалить выбранную фигуру.

### 7.3 Save / cancel
- Cmd-S или Save → записывает PNG обратно в shot1/shot2.
- Cancel сбрасывает изменения.
- Undo/Redo (Cmd-Z / Cmd-Shift-Z) внутри редактора отдельный стек.

## 8. Чат и Presence

### 8.1 Чат
- Inline под Before/After, всегда видим.
- "Type a message and press Enter…" — Enter отправляет, Shift+Enter — перенос.
- Каждое сообщение: аватар инициал + ник + текст. Hover — крестик удаления (только своё).
- В Present mode чат с прокруткой и кнопкой "+ Add screenshot".

### 8.2 Realtime
- Supabase Realtime канал `task:<id>` на каждой открытой задаче.
- Новое сообщение прилетает другим подписчикам < 1 с.
- Закрытие чата отписывает.

### 8.3 Presence
- В Present mode справа от Assignee — стек аватаров других людей, кто сейчас смотрит эту же задачу.
- Появляются/исчезают в реальном времени.
- При Next/Prev переключается канал.

## 9. Ссылки (Drive integrations)

### 9.1 Добавление
- Поле "Paste Google Drive link…" в карточке задачи.
- При вставке URL: detect type (Doc / Sheet / Slide / PDF / generic) → favicon + auto-title через `gdriveTitleFromUrl`.
- Enter добавляет.

### 9.2 Список
- Иконка-фавикон + название.
- **Single-click по названию = открыть ссылку** (внешний браузер).
- **Double-click или иконка-карандаш = войти в режим переименования** (contenteditable).
- Enter сохраняет, Escape откатывает, blur коммитит.
- Если пустое имя при blur — fallback на gdriveTitleFromUrl, тост "Link title reset to URL".
- Иконка-крестик справа — удалить (с undo).

### 9.3 Drag-reorder
- Drag-handle `⋮⋮` слева; drop на другую ссылку — переупорядочивает.
- Подсветка drop-target — синий wash + dashed outline (в dark-mode ярче, 18% alpha).
- `onLinkDragEnd` сбрасывает drag-state даже при дропе мимо.

## 11. Undo / Redo

### 11.1 Покрытие
- Create / Delete / Duplicate task
- Change status / assignee / deadline / pin / archive
- Reorder задачи (drag-drop)
- Rename / reorder / delete ссылок
- Chat-message create / delete
- Bulk-операции — одной записью

### 11.2 UX
- Cmd-Z / Cmd-Shift-Z (Ctrl на Windows).
- Тост "Undid: Change status" появляется при undo.
- Стек не должен срабатывать на программные изменения (обёртка `_runWithoutUndo`).

## 12. Present mode

- Открывается двойным кликом по карточке или через меню.
- Левая часть: Before/After крупно + миниатюра старого скрина.
- Правая часть: Chat + Notes.
- Шапка: Status pill / Assign / Deadline chip / Presence stack / X.
- Низ: Prev / N of M / Next / Add screenshot.
- Esc или X — закрывает.
- Cmd-стрелки = Prev/Next.
- При смене задачи через Next/Prev — никакого мерцания deadline-чипа (renderDeadline всегда в pres-head, см. v0.1.158 P0-2).

---

## 13. Тёмная тема

- Кнопка-переключатель внизу сайдбара "Dark mode".
- Сохраняется в LocalStorage, переживает перезапуск.
- Все surfaces, текст, иконки, drop-target подсветки — отдельные значения для тёмной темы.
- Скрины и аннотации не инвертируются.

## 15. Облачная синхронизация

### 15.1 Что синхронизируется
- Все задачи / проекты / чаты / ссылки / шаблоны / шифровка.
- Конфликты разрешаются по `updated_at` (last-write-wins).

### 15.2 Офлайн
- Все изменения логируются локально и пушатся при подключении.
- Индикатор `cloud / offline` в шапке.

### 15.3 Realtime
- Project-level канал — обновления чужих задач прилетают в реальном времени.

## 18. Нефункциональные требования

### 18.1 Производительность
- Холодный старт < 2 с до интерактивности (Tauri).
- Рендер 500 задач в одном проекте — без заметного лага.
- DOM-патчинг (`patchPresent`) — без мерцания при изменении одной задачи в Present.

### 18.2 Надёжность
- Crash в редакторе аннотаций не должен ронять основное окно.
- Потеря интернета — мгновенный fallback на local, никаких диалогов.
- Закрытие приложения с несинкнутыми изменениями — push при следующем старте.

### 18.3 Безопасность
- Public link не должен раскрывать соседние задачи проекта.
- viewer не должен иметь UI для write-операций (вне зависимости от RLS).
- Все Supabase-запросы — через RLS, никакого service_role в клиенте.

### 18.4 Доступность (a11y)
- Все интерактивные кнопки — с `aria-label`.
- Toggle-кнопки (Archive, Pin) — с `aria-pressed`.
- Контраст текста на surface — не ниже AA (4.5:1).

## 19. Acceptance criteria тесткейсов (что проверяет QA на каждой сборке)

### 19.1 Smoke (5 минут, на каждом релизе)
1. Логин проходит, проекты грузятся.
2. Создать новый проект — Enter работает как Create.
3. Создать задачу, переключить статус, удалить, Undo восстанавливает.
4. Сделать скрин, открыть редактор, нарисовать стрелку, сохранить.
5. Открыть Present, переключить Next, закрыть.

### 19.2 Регресс-сюита (~45 минут, на больших релизах)
- Все 8 статусов прорисованы корректно.
- Drag-drop задач внутри недели, между неделями, на пин.
- Archive toggle: иконка, aria-pressed, opacity ≥ 0.9 у Unarchive-кнопки.
- Pin / Unpin: задача всплывает / уходит обратно.
- Deadline: добавить, изменить через Present, очистить.
- Ссылка: добавить, single-click открывает, dblclick/pencil переименовывает, drag-reorder работает.
- Чат: написать, удалить своё, увидеть чужое < 1 с (с двух машин).
- Public link: скопировать, открыть в инкогнито, viewer не видит write-кнопок.
- Dark mode: переключить, все surfaces корректные.
- Offline: выключить wifi, изменить задачу, включить — синк.
- Editor: все 11 инструментов (V H R C A L P T B Q M), shift-констрейн, Esc, Delete.
- Шорткат D **не должен** активировать никакой инструмент.

### 19.3 Cross-browser / cross-OS
- macOS WKWebView: фокус-репэйнт после смены окон (см. EngiBoard_WKWebView_Repaint_Fix).
- Windows WebView2: drag-drop файлов работает.
- Веб в Chrome / Edge: то же поведение.
